1.1 What is Programming?

Programming is the art of making a computer do what you want it to do. When trying to understand what "programming" means, it is also important to first define the term "algorithm". An algorithm is a detailed step-by-step formula for solving a problem or completing a task. Computer Programming is essentially the art of writing algorithms that tell a computer how to perform a task.

Programming is not synonimous to computer science. Computer science is more than just programming, although it is a critical part of a computer scientist's work.

So what is an Algorithm?

This recipe has the following characteristics which is what defines an algorithm:

  • It has a sequence of simple steps
  • It applies a process that specifies when each step is executed.
  • It has a means of determining when to stop.

Programming Languages

To "speak" to a computer, you need to use a language it understands. For instance, if grandma only speaks Kiswahili it's pointless to try having a conversation in English with her. Computers only understand a language called binary. "Binary" just means two in simple English. So when we say a computer only understands a "binary language", it means the way it represents numbers is just in form of ones (1s) and zeros (0s). So if for example you want to write a name like "Moses", a computer doesn't understand that. The name Moses in binary is some number of 1s and 0s. We will get into details in Chapter 5. For now it's just good to know that computers understand binary number representation.

1.3 Introduction to the Terminal

A terminal is a program on a computer that executes commands. You can tell it to delete files, launch programs, run code, and even "hack" computers (pun-intended). It is also referred to as the command line, interpreter, or shell. You can't use your mouse when working from the terminal. You might now be asking, what are these commands that a shell executes?

What is a command?

A command is just a statement that you tell a computer to execute. Here are some of the most commonly used and useful terminal commands:

ls: lists all files or directories in the current directory

pwd: prints the current working directory (displays current directory)

cd new_directory: change directory from current working directory to new_directory

cd ..: change to parent directory (move up a directory)

mkdir new_directory: make a directory called new_directory

mv file_name new_directory: move a file called file_name to a directory called new_directory

rm file_name: remove file called file_name from current directory

1.4 Using Git

To send your code to an online storage site, you need some form of "transport". That is what git is. It keeps track of what you want to "transport", and you can tell it where you want to send your code. It's like a truck. You add stuff. Then tell the driver where to go, and send it when everything is ready. You will need Git throughout this course and throughout your life as a computer programmer. Before we download and install git, let's first go ahead and create a Github account. Github is just a website where programmers store their code.

Creating a Github account

Go to the Github homepage and create an account. Github is just like any other social network like Facebook or Instagram. You can join a group on Github, follow someone, upload your project, or share other projects you might find to be interesting. You can share anything on Github - your pictures, files, or code. What is amazing about using Github is that you can use it without knowing any code at all. Git is the software at the heart of the Github "social network". We shall be using some terms which we should define first before we start:

  • Repository: This is just a storage space where you put your projects. Sometimes its just shortened to "repo". You can store files on your computer or you can store them online at Github. When you store them at Github, the storage space there is referred to as your "Online Repository". It can also be referred to as the "remote". When you store them on your computer, that is referred to as your "Local Repository".
  • Branch: Sometimes many people might be working on the same project and all of them are storing the code for the project in the same Online Repository. It can get confusing, so you need to ensure who did what and which part of the project they contributed in. To avoid confusion, the project can be done in different "sections". In the end, all these "sections" are brought together when the project is finished. We call all the other sub-sections branches and the main one where they are all brought together is called the "master branch'. Git allows you to create different branches for your project. We will explore this more as we go on.

Setup your name and email. Open your terminal and type this command to setup your name then hit "Enter". Remember you can't use your mouse when working from the terminal.

git config --global user.name "Your Name"

Type this to setup your email.

git config --global user.email "your_email@something.com"

Awesome! We have successfully downloaded and installed Git. We can now go ahead to create our Local Repository.

Creating your Local Repository

We will use git to create our local repository and connect it to the Online Repository we created earlier. For the sake of clarity, we will store our files on the Desktop. We will need to create a folder in our Desktop where we will be storing all our work for this course. As we said earlier, you can do anything from your terminal. You can go to the desktop and create a folder for your project without leaving the terminal. Let's do that:

Open your terminal and type this command and hit enter. This will tell your terminal to start operating from your Desktop.

cd ~/Desktop

Next, we need to create a folder for our project. We will name this folder, "cs101" without the quotes. Type this command on your terminal and hit enter.

mkdir cs101

"mkdir" essentially means "make directory" and "cd" means "change directory" in the language that your computer understands. It won't understand if you typed that in full. Our terminal is still operating from the Desktop, so we need to "change directory" and work from inside our "cs101" folder. To do that run this command (run just means typing it to your terminal and hit enter)

cd cs101

Wow, we are almost there. We can now just go ahead and create our Local Repository. Run this command from your terminal. "Init" means that we are just initializing an empty Git repository. There is nothing in our folder yet.

git init

Let's put something in our folder and see if we can send it to our Online Repository at Github. You can use your terminal to create a file called "readme.txt". To create that file run this command

touch readme.txt

Next, we need to add this file to our truck before we transport it to the destination which is our Online Repository. Our truck, as we said earlier, is Git. To add this file to git, run this command

git add readme.txt

So our file is in now on the "truck", but we need some way to tell the "driver" that the truck is full or that there is nothing more to add. We call this "commit". After we have made all our changes to our files, we commit them to Git before we send them to our Online Repository. Run this command from your terminal.

git commit -m "first commit"

Before we send our file to the Online Repo, we need to tell git where the online repo is. There are so many addresses online, it's just like telling someone who has never been to Kenya to drive a truck to Isiolo. Remember we also call our Online Repo the "remote". Let's just go back to our online repository we created and copy the address. Notice the line with remote in it? Copy that and past it to your terminal.


git add remote origin https://github.com/your_username/repo_name.git

Finally, we can now go ahead and send our file to Github. We call this "pushing". To push to Github, run this command


git push -u origin master

Fantastic! Let's go to Github and see if the file is there. Just refresh your page, and you should see your file there.
