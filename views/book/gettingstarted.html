1.1 What is Programming?

Programming is the art of making a computer do what you want it to do. When trying to understand what "programming" means, it is also important to first define the term "algorithm". An algorithm is a detailed step-by-step formula for solving a problem or completing a task. Computer Programming is essentially the art of writing algorithms that tell a computer how to perform a task.

Programming is not synonimous to computer science. Computer science is more than just programming, although it is a critical part of a computer scientist's work.

So what is an Algorithm?

It has been developed at the University of California, Berkeley and is used to teach an introductory course, CS 50.

This recipe has the following characteristics which is what defines an algorithm:

  • It has a sequence of simple steps
  • It applies a process that specifies when each step is executed.
  • It has a means of determining when to stop.

Programming Languages

To "speak" to a computer, you need to use a language it understands. For instance, if grandma only speaks Kiswahili it's pointless to try having a conversation in English with her. Computers only understand a language called binary. "Binary" just means two in simple English. So when we say a computer only understands a "binary language", it means the way it represents numbers is just in form of ones (1s) and zeros (0s). So if for example you want to write a name like "Moses", a computer doesn't understand that. The name Moses in binary is some number of 1s and 0s. We will get into details in Chapter 5. For now it's just good to know that computers understand binary number representation.

1.3 Introduction to the Terminal

A terminal is a program on a computer that executes commands. You can tell it to delete files, launch programs, run code, and even "hack" computers (pun-intended). It is also referred to as the command line, interpreter, or terminal But you can't use your mouse. You might now be asking, what are these commands that a shell executes?

What is a command?

If you've ever join a scouts or girl-guides club, you know what to do when the "commander" tells you to do something. When using a shell, you are essentially the commander and the computer obeys your commands. We will go ahead and look at some useful commands you can start using right away.

1.4 Using Git

To send your code to an online repository, you need some form of "transport". That is what git is. It keeps track of what you want to "transport", and you can tell it where you want to send your code. It's like a truck. You add stuff. Then tell the driver where to go, and send it when everything is ready. Ok, departing from this metaphor. Let's now go ahead and install git so we can see what we can do with it. You will need it throughout this course and throughout your life as a computer programmer. But first we need to create a Github account. Github is just a website where programmers store their code.

Creating a Github account

Go to https://github.com and create an account.

Setup Git

Download the right git version for your computer here. Windows Linux Mac

Setup Name and Email

git config --global user.name "Your Name"

git config --global user.email "your_email@something.com"

Creating a Git Repository

git init

Add some files to your repo

touch readme.txt

git add readme.txt

git commit -m "first commit"

Connect to a Remote

So now you have loaded your "truck" with a file called readme.txt, and you want to tell the driver where the destination is. To do that you need to go to your github account and create a new repository.

Copy the url of the repository you just created and go back to your terminal. Run this command

git remote add origin https://github.com/your_username/repo1

Awesome! What is now left is just sending the file to the destination. We do that by running the following command.

Pushing to Github

git push origin master

Understand what we have just done

Let's now look at what we have just done step by step and understand everything.

Checking Status

Sometimes you need to confirm is you've send your files to github. Git allows you to do so. Or there's a file that you didn't add to your "truck" - that is an untracked file. If it is untracked, you need to add by running git add nameoffile or if there are many files you can add them all simultaneously using git add . . Notice the dot after the space. It's not a mistake. To check the status of your "commits", run this command from inside your git repository (you will get some error or incorrect stuff if you do it elsewhere)

git status
